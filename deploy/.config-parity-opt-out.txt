# Config parity opt-out
#
# Variables listed here are intentionally absent from the Helm values.yaml and
# Kubernetes configmap.yaml.  Each entry carries a short justification so
# reviewers understand the deliberate gap.
#
# Format:  VAR_NAME  — reason
#
# Wave-F (enterprise hardening) additions ─────────────────────────────────────
#
MCP_AUDIT_DURABILITY               — operator tunable at runtime; safe default
                                     (best_effort) ships in the base image.
                                     Helm operators who want fail_closed add it
                                     as a plain env var override in their values.
MCP_HTTP_LEGACY_POLICY             — only relevant when MCP_TRANSPORT=http.
                                     New deployments use streamable_http.
                                     Opt-in by setting in values.extraEnv.
MCP_HTTP_INLINE_METRICS_ENABLED    — off by default; side-channel
                                     MCP_METRICS_BIND is the recommended path
                                     for shared-service metric scraping.
MCP_HTTP_INLINE_METRICS_AUTH_MODE  — depends on MCP_HTTP_INLINE_METRICS_ENABLED;
                                     not wired into Helm/k8s until the feature
                                     is explicitly adopted.
MCP_HTTP_INLINE_METRICS_BEARER_TOKEN — secret material; must be injected via
                                     a Secret/ExternalSecret, not a ConfigMap.
                                     Operators add it to values.extraSecretEnv.
ENVIRONMENT                        — usually set via a constant or a different
                                     mechanism in CI/CD; not a primary runtime
                                     config for Helm/k8s users today.
MCP_ALLOW_DEV_BACKEND              — safety guard against using file/memory in
                                     prod; excluded from templates to nudge
                                     users toward Postgres for k8s.
MCP_CONTROL_PLANE_AUDIT_CAP        — advanced tuning; use extraEnv if needed.
MCP_CONTROL_PLANE_AUDIT_RETENTION  — advanced tuning; use extraEnv if needed.
MCP_OIDC_VERIFY_CACHE_TTL          — advanced tuning; default is safe.
CLOCKIFY_TOOL_TIMEOUT              — advanced tuning; default is safe.
MCP_HTTP_MAX_BODY                  — deprecated legacy fallback for MCP_MAX_MESSAGE_SIZE
CLOCKIFY_DELTA_FORMAT              — advanced tuning; default "merge" is safe.
MCP_PROFILE                        — invocation-time selector for the five
                                     canonical deployment shapes. Helm users
                                     typically override individual values
                                     rather than delegate to a profile;
                                     operators who do want a profile add it
                                     via values.extraEnv (same pattern as
                                     ENVIRONMENT above). See ADR-0015.
MCP_FORWARD_AUTH_TRUSTED_PROXIES   — security gate added 2026-04-30 per
                                     ChatGPT audit. Defaults to empty (legacy
                                     trust-everything posture for self-hosted
                                     installs). Hosted profiles + doctor
                                     --strict refuse to start without it,
                                     so operators add it to values.extraEnv
                                     when wiring forward_auth behind a known
                                     proxy. Pattern matches MCP_PROFILE /
                                     ENVIRONMENT — operator-supplied at
                                     deploy time, not a templated default.
MCP_BEHIND_HTTPS_PROXY             — security gate added 2026-04-30 per
                                     ChatGPT audit. Default 0 keeps HSTS off
                                     on plaintext responses. Operators behind
                                     an HTTPS-terminating proxy set =1 via
                                     values.extraEnv. Templating a default
                                     here would silently flip on HSTS for
                                     every Helm install regardless of
                                     ingress posture.
